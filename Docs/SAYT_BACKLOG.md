# Sayt — 포트폴리오 이후 개선 백로그

현 포트폴리오(Sayu) 완성 이후, 동일 코드베이스 위에서 진행할 개선 프로젝트의
백로그. "프레젠테이션 코어"(영상/지원 품질에 직결)와 "심화 백로그"(지원과
병행 가능한 장기 과제)의 구분은 로드맵 재편 세션에서 확정한다.

## 전투/캐릭터
1. **캐릭터 로코모션 고도화** — 이동/점프/달리기/대시 + 애니메이션 전환.
   Paragon: Aurora 애니메이션 세트 활용 (현재: 블렌딩 없음)
2. **Aurora 스킬킷 전체 구현** — 고유 스킬 전체를 GAS 어빌리티로, Niagara/사운드 FX 연동
3. **전투 디버그/치트 툴** — 전투 시각화 토글 포함 (Phase 3에서 이연)

## UI
4. **중앙화 UI Manager** — CommonUI Activatable Widget Stack + Input/Action Router(Lyra 방식)
   + 공식 MVVM 플러그인. 현재 인벤토리/HUD의 AddToViewport 직접 부착(스택 미인지)을
   정식 스택 push 구조로 교체. 목표는 실무 투입 가능한 깊이
5. **체력바 확장 기획** —
   - 보스: 화면 상단 대형 바, 줄(라인) 체력(xN 카운터, 줄마다 채움/배경 색 페어 스왑),
     수십억 단위 수치 텍스트, 전투 해제 시 숨김.
     현재 SSayuHealthBar는 "퍼센트 + 색 2개"만 아는 구조로 이미 이 요구에 정렬됨 —
     Health→(줄 인덱스, 줄 내 %, 색 쌍) 매핑은 위젯 밖 게임 로직 담당
   - 일반 몬스터: 머리 위 소형 바, 거리 기반 스케일링(RenderTransform), 플레인 스타일

## 인벤토리
6. **부분 겹침 아이템 스왑** (Tarkov식 드래그)
7. **비정형 footprint + 회전** — GridSize를 셀 점유 마스크로 재설계 (회전은 비정형과 번들)
8. **아이템 액션 컨텍스트 메뉴** — 사용/버리기/스택 분할 등 실제 액션 구현 이후

## 인프라
9. **독립 Launcher** — 매니페스트 대조 버전 체크 + 업데이트 다운로드 + 본게임 실행.
   게임 프레임워크 없는 bare Slate Application (AssetManager Chunking과는 별개 시스템,
   chunk 매니페스트를 델타 다운로드에 활용 가능)
