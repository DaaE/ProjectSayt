# Sayt — 포트폴리오 이후 개선 백로그

현 포트폴리오(Sayu) 완성 이후, 동일 코드베이스 위에서 진행할 개선 프로젝트의 백로그.

## 운영 방식 (2026-07-18 확정)

- **인박스**: 새 아이디어는 판정 없이 한 줄로 즉시 기록. 판정은 세션 경계(Stage 종료 등)에서 일괄.
- **판정 4축**: ① 채용 신호(이력서 한 줄/면접 질문이 되는가) ② 영상 노출 ③ 비용 ④ 기존 시스템 시너지
- **등급**: 프레젠테이션 코어(지원 시작선 포함 — 정본은 `ROADMAP.md`로 이관됨) /
  심화 백로그(지원 후 병행 축적) / 아이스박스(취업 신호 미미, 죄책감 없이 방치)
- 구 백로그 1·2번(로코모션, Aurora 스킬킷)은 프레젠테이션 코어로 승격되어 ROADMAP.md로 이동.

## 심화 백로그 (우선순위 순)

1. **UI 프레임워크** — CommonUI Activatable Widget Stack + Input/Action Router(Lyra 방식)
   + 공식 MVVM 플러그인 + **데이터 주도 설정 시스템**(앵커 결과물).
   - 설정 시스템: Lyra GameSettings의 레지스트리 패턴 참고 단순화 자체 구현 —
     설정 항목 정의(데이터) 추가만으로 UI 카테고리·위젯 자동 생성, 값 변경→적용→저장 파이프라인.
     그래픽(GameUserSettings/Scalability/DLSS 공식 플러그인) → 키(EnhancedInputUserSettings) →
     오디오 순 증분 확장
   - 현재 인벤토리/HUD의 AddToViewport 직접 부착(스택 미인지)을 정식 스택 push로 교체,
     `USaytUIInputModeSubsystem`(가교)도 이때 정리
   - 목표는 실무 투입 가능한 깊이
2. **목표 추적 커널 + 튜토리얼 프레젠테이션** *(1번 종속)* —
   - 커널: 목표 정의(DataTable/DataAsset: 트리거 GMS 태그·조건·횟수) + 진행 매칭(GMS 순수
     리스너, 게임플레이 코드 무수정) + 완료 저장(SaveGame). 퀘스트-튜토리얼 공통 코어를
     분리하는 온라인 게임 실무 구조의 로컬 재현
   - 튜토리얼 레이어(소비자 #1): 이벤트 트리거 → 시간 정지 → 암영+대상 스포트라이트(Slate
     컷아웃 오버레이) → 입력 게이팅(UI 라우팅 + GAS 태그 화이트리스트 2층) → 완료 시 진행.
     시연용 2~3스텝
   - 퀘스트 콘텐츠(체인/보상/저널)는 아이스박스 — 만들지 않는 것이 스케일 통제선
3. **넷코드 직접 구현** — Phase 10 흡수분(이론 비교·Prediction 심화·lag compensation·서버
   권위) 이후의 실구현 확장
4. **전투 디버그/치트 툴** — 전투 시각화 토글 포함 (Phase 3에서 이연)
5. **체력바 확장 잔여** — Phase 8 Stage 2가 줄 체력 보스 바·몬스터 패널을 소화한 뒤 남는 것:
   보스 바 텍스처 장식 룩(디아블로4/PoE2급 — 아트 소스 필요, "에셋 경로"), 수십억 단위 수치
   텍스트, 전투 해제 시 숨김 연출
6. **타격감 확장** — 넉백, 랙돌, 최소 패스(히트스톱/셰이크/플래시 — Phase 11~12로 당김) 이후분
7. **부분 겹침 아이템 스왑** (Tarkov식 드래그)
8. **비정형 footprint + 회전** — GridSize를 셀 점유 마스크로 재설계 (회전은 비정형과 번들)
9. **아이템 액션 컨텍스트 메뉴** — 사용/버리기/스택 분할 등 실제 액션 구현 이후
10. **독립 Launcher** — 매니페스트 대조 버전 체크 + 업데이트 다운로드 + 본게임 실행.
    게임 프레임워크 없는 bare Slate Application (AssetManager Chunking과는 별개 시스템,
    chunk 매니페스트를 델타 다운로드에 활용 가능)
11. **로컬라이징 시스템** — 메뉴 언어 설정 연동. 단 **FText 규율(유저 노출 문자열은 FText)은
    지금부터 즉시 적용** (CONVENTIONS.md 수록) — 소급 비용 회피용 사전 정지작업
12. **Mass Entity(ECS 대규모 군중)** — 정본은 ROADMAP.md Phase 13(선택). 착수 시점 재벤치마크

## 아이스박스

- 텍스트 컷씬(선택지 분기) — 데이터 주도 대화 시스템, 액션 포폴과 장르 결 상이
- 채팅창 — 언리얼에 기본 채팅 UI 위젯 없음(직접 제작 필요), 우선순위 낮음
- 풀 퀘스트 콘텐츠(체인/보상/저널) — 커널(심화 2번)만 만들고 콘텐츠는 만들지 않음
